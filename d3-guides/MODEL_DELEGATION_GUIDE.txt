===========================================================================
MODEL DELEGATION GUIDE FOR THE META VIEWS STACK
===========================================================================

This guide recommends which model to use for each toolkit workflow based on
the kind of work involved.

Stable delegation logic:
- Use GPT for exploration, inventories, and planning
- Use Codex for bounded implementation and pattern-based transformation
- Use a standards-oriented model for accessibility and written review quality
- Use a deeper judgment-oriented model for high-stakes launch decisions

---------------------------------------------------------------------------
WHEN TO USE EACH MODEL
---------------------------------------------------------------------------

GPT-5.4
  - project planning and schema definition
  - session bootstrap and file-tree mapping
  - review inventories and structured audits
  - design-token and naming-rule analysis

GPT-5.4 Mini
  - cheaper bootstrap passes on small or tidy projects
  - lower-risk restore-point check and create operations

GPT-5.3 Codex
  - generating or cleaning bounded PHP, Twig, CSS, and mbjson artifacts
  - pattern-based transformations after the design or audit decision is clear

Claude Sonnet 4.6
  - accessibility review
  - standards-heavy written audit output

Claude Opus 4.6
  - final judgment on harder performance or launch-risk questions
  - situations where missing a serious issue would be costly

---------------------------------------------------------------------------
REASONING LEVEL GUIDE
---------------------------------------------------------------------------

LOW
- mechanical cleanup
- repetitive transformations

MEDIUM
- implementation from a clear plan
- review work with limited ambiguity

HIGH
- bootstrap and discovery work
- schema or workflow design
- launch-readiness reviews

EXTRA HIGH
- large or unusually tangled site projects
- broad audits where the first pass must be especially complete

Default rule:
- Start with GPT-5.4 at High when you are still mapping the work.
- Drop to Codex once the change is bounded.
- Escalate to stronger judgment models for accessibility or high-stakes launch calls.
